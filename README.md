# m-online-archive

## 项目简介

`m-online-archive` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为存放静态 HTML 页面及其相关资源的通用空间，便于长期保存与访问。

## 目录说明

```
m-online-archive/
├── pages/          # 存放各个独立的 HTML 页面及附属资源
├── assets/         # 页面共享的样式、脚本、图片等公共资源
├── index.html      # 仓库首页，可快速浏览已归档页面列表
└── README.md       # 本文件
```

各目录功能如下：

- **pages/**：每个子文件夹代表一个独立的归档页面，文件夹内包含 HTML 文件及相关资源（如 CSS、JS、图片等）。
- **assets/**：存放多个页面可能共用的资源文件，例如基础样式、通用脚本或占位图片。
- **index.html**：仓库入口页面，提供对已归档页面的索引与导航。

## 页面归档说明

本仓库中每个归档页面均为独立的 HTML 文档，结构清晰，不依赖外部服务。归档内容可能包括：

- 静态网页快照
- 离线可用的交互式页面
- 其他需要长期保存的 HTML 资源

归档的页面均经过整理，确保在本地或静态托管环境中均可正常浏览。若需添加新页面，请遵循以下规范：

1. 在 `pages/` 下创建以页面名称命名的文件夹。
2. 将 HTML 文件及所有关联资源放置于该文件夹内。
3. 若页面引用公共资源，请使用相对路径指向 `assets/` 目录。
4. 更新 `index.html` 中的索引列表以包含新页面。

## 维护说明

本仓库由维护者不定期更新。归档页面的添加、修改或移除会记录在 Git 提交历史中，以便追溯变更。

- 欢迎通过 Issue 提出归档建议或报告链接失效（若存在引用外部资源的情况）。
- 提交 Pull Request 前请确保页面资源完整、路径正确，并遵循上述目录规范。
- 仓库不保证所有归档页面的长期可用性，但会尽力保持其可访问状态。
